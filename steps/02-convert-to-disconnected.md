# Step 2: Convert a Connected Cluster to Disconnected

In this step, we'll convert an OpenShift cluster from **connected** to **disconnected** mode so it pulls images from your local Quay mirror-registry.

Follow [Red Hat's documentation](https://docs.redhat.com/en/documentation/openshift_container_platform/4.19/html-single/disconnected_environments/index#connected-to-disconnected-config-registry_connected-to-disconnected) for reference.

---

## 1. Back Up the Pull Secret

```bash
$ oc get secret pull-secret -n openshift-config -o yaml > pull-secret-backup.yaml
```

## 2. Create a Config File for Your Destination Registry

```json
{
  "auths": {
    "<MIRROR_REGISTRY>": {
      "auth": "<YOUR_BASE64_AUTH_TOKEN>",
      "email": ""
    }
  }
}
```

> **Note:** Generate the base64 auth token with:
> ```bash
> $ echo -n 'username:password' | base64
> ```

## 3. Update the Pull Secret

```bash
$ oc set data secret/pull-secret -n openshift-config \
    --from-file=.dockerconfigjson=config.json
```

## 4. Verify the Pull Secret Was Updated

```bash
$ oc get secret pull-secret -n openshift-config -o yaml
```

Confirm the `.dockerconfigjson` field contains your registry entry. You can decode and verify:

```bash
$ oc get secret pull-secret -n openshift-config -o jsonpath='{.data.\.dockerconfigjson}' | base64 -d
```

## 5. Download the Registry Certificate

Download your Quay registry's PEM certificate and create a ConfigMap:

```bash
$ oc create configmap registry-config \
    --from-file=<MIRROR_REGISTRY>.pem=<REGISTRY_CERT> \
    -n openshift-config
```

> **Tip:** Use the double-dot format (`mirror.example.net..8443`) as the key name — OpenShift replaces the dots with a colon internally.

## 6. Patch the Image Configuration

Add the trusted CA reference to the cluster's image config:

```bash
$ oc patch image.config.openshift.io/cluster \
    --patch '{"spec":{"additionalTrustedCA":{"name":"registry-config"}}}' \
    --type=merge
```

Verify:

```bash
$ oc get image.config.openshift.io/cluster -o yaml
# Look for:
spec:
  additionalTrustedCA:
    name: registry-config
```

## 7. Disable Default Operator Sources

Disable all default OperatorHub sources via the CLI:

```bash
$ oc patch OperatorHub cluster --type json \
    -p '[{"op": "add", "path": "/spec/disableAllDefaultSources", "value": true}]'
```

You can also do this via the UI: **Administration → Cluster Settings → Configuration → OperatorHub → Cluster → Sources**.

## 8. Apply Mirror Resource Files

Navigate to the cluster resources directory generated by `oc-mirror`:

```bash
$ cd <CACHE_DIR>/oc-mirror/mirror1/working-dir/cluster-resources
```

Apply the YAML files:

```bash
$ oc apply -f .
```

> **Important Notes:**
> - There are **two** `signature-configmap` files (`.json` and `.yaml`). Apply only **one** of them.
> - The `updateService.yaml` file should only be applied if you're hosting the **OpenShift Update Service** on this cluster. If not, you'll see an error like `resource mapping not found for kind "UpdateService"` — this is expected.

Expected output:
```
clustercatalog.olm.operatorframework.io/cc-redhat-operator-index-v4-19 created
catalogsource.operators.coreos.com/cs-redhat-operator-index-v4-19 created
imagedigestmirrorset.config.openshift.io/idms-release-0 created
imagedigestmirrorset.config.openshift.io/idms-operator-0 created
imagetagmirrorset.config.openshift.io/itms-generic-0 created
imagetagmirrorset.config.openshift.io/itms-release-0 created
configmap/mirrored-release-signatures created
```

## 9. Verify the Setup

Check that the image mirror sets are installed:

```bash
$ oc get imagedigestmirrorset
NAME              AGE
idms-operator-0   76s
idms-release-0    76s
```

Verify they were created by oc-mirror:

```bash
$ oc get imagedigestmirrorset -o jsonpath='{.items[?(@.metadata.annotations.createdBy=="oc-mirror v2")].metadata.name}'
idms-operator-0
idms-release-0
```

Check the ImageTagMirrorSets:

```bash
$ oc get imagetagmirrorset -o jsonpath='{.items[?(@.metadata.annotations.createdBy=="oc-mirror v2")].metadata.name}'
itms-generic-0
itms-release-0
```

Check the CatalogSource:

```bash
$ oc get catalogsource -n openshift-marketplace
NAME                             TYPE   AGE
cs-redhat-operator-index-v4-19   grpc   106s
```

Check the ClusterCatalog:

```bash
$ oc get clustercatalog
NAME                             LASTUNPACKED   SERVING   AGE
cc-redhat-operator-index-v4-19                   118s
```

## 10. Test the Operator Installation

Install an operator (e.g., `odf-operator`) to confirm it pulls from your local registry. You can also verify release images are being used by scaling up a node and checking the pod images.
